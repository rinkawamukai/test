■ IFTTT Applet
IF：RSS → New feed item
    Feed URL: https://rinkawamukai.github.io/test/public/feed.xml

THEN：Google Calendar → Quick add event
      Quick Add Text:
      {{EntryContent}} at {{EntryPublished}}

■ RSS 仕様
pubDate      → 前日21:00（RFC-822）
description  → 本来の予定時刻（例：2026/04/02 08:39）



1. Applet を作る

IFTTT を開く → Create を押す
If This をクリック


✅ 2. “RSS Feed” を選ぶ

サービス検索で RSS を検索
トリガー：New feed item を選択（新しい <item> が追加された時だけ発動）
→ IFTTT は Feed をポーリングし、RSS の以下の材料を提供してくれる：

EntryTitle
EntryContent（＝RSS <description>）
EntryPublished（＝RSS <pubDate>：RFC-822日時） [ifttt.com]



フィード URL に入力：
https://rinkawamukai.github.io/test/public/feed.xml


✅ 3. “Then That” で Google Calendar を選ぶ

Google Calendar と入力して選択
アクションは Quick add event を選ぶ

Quick Add は“自然言語の日時”を解析して予定を自動生成する仕組み。 [ifttt.com]




✅ 4. Quick Add Text を入力
RSS の構成は：

pubDate → 前日21:00
description → “本来の予定時刻（例：2026/04/02 08:39）”

なので、Google カレンダーにはこう伝える：
{{EntryContent}} at {{EntryPublished}}

これにより…


カレンダーの イベント時刻
→ EntryPublished（＝前日21時）


イベントの タイトル（先頭部分）
→ {{EntryContent}}（＝本来の予定時刻の文字列）


イベントの 通知
→ Googleカレンダーの通知設定（例：開始0分前）がそのまま発火 → 前日21時に通知
（Googleカレンダーは通知をイベント開始時間の相対時刻で出すため） [stackoverflow.com]



✅ 5. キャンペーン：Google Calendar 通知設定を確認
Google カレンダーの設定（ブラウザ版）で、該当カレンダーの：

「通知（Event notifications）」
で、
通知1件：0分前（または 1分前など）
を設定しておけば、前日21:00 にピッタリでスマホ通知が鳴る。


✅ 6. 動作確認の手順

RSS（feed.xml）に新しい <item> が生成されているか確認
IFTTT → “Check now” をクリック（ポーリングを待たずに即確認）
Google カレンダーにイベントが入る
イベントの開始時刻＝「前日21:00」になっていることを確認
スマホで通知が鳴くか確認（通知設定が必要）
